1. Admin Control Panel (Change Password, Edit ID, Level, Post, Name)
User Management System: Create a table (users) with columns: id, name, password, level (e.g., Admin/Core/Basic), post, active_status.
Edit Functionality: Only users with level='Admin' can access the update_user.php form to change data.
Password Change: Password fields should be encrypted (password_hash in PHP or similar).
2. Login/Logout Log (Entry and Exit Time)
Audit Table: Create a login_log table: log_id, user_id, login_time, logout_time, session_id.
Logging:
Entry: On login, record user_id and NOW() into login_time.
Exit: On logout, update the row matching session_id with NOW() in logout_time.
3. Active Events & Registration
Connection: Active events in the database are queried to populate the Event Page.
Registration/Attendance: A registration form linked to event_id is required. Include:
Dropdown to select events.
Fields for user details (Name, ID, etc.).
Attendance Check-in: A button for users to "Check In" or a scanned QR code to record attendance.
4. Publish/Unpublish Button
Form Control: A status column (boolean: 0 for Unpublished, 1 for Published) in the events table.
Logic:
If Published (1): Form appears.
If Unpublished (0): Render a message: "We will be Back Soon with a new event."
5. Equipment Section (With Sports & Emojis)
Database: A types table containing equipment names and emoji characters (e.g., "Football ⚽", "Basketball 🏀", "Tennis 🎾").
Display: Dynamically load these from the database to ensure all items and emojis are shown.
6. Role-Based Access (Basic Level Limitations)
Session Management: Check user level upon login.
Basic User View: A restricted dashboard (basic_dashboard.php) that only queries user_position and active_game from the database.
Hidden Features: Admin/Core UI elements are hidden if user.level != 'Core' or 'Admin'.
7. Communication Section (Core to Basic Level)
Messaging System: A messages table: message_id, sender_id, receiver_id (or group_type), message_text, timestamp.
Interface: A chat or bulletin board interface where 'Core' users can post, and 'Basic' users can only view.
